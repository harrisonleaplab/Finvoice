            <h1 class="text-5xl font-bold display-font mb-4">Security Overview</h1>

            <p class="text-slate text-sm mono-font mb-12">How AdvisorNote ensures compliance and data safety.</p>


            <div class="space-y-12">

                <div class="bg-white p-8 border-2 border-charcoal">

                    <h2 class="text-3xl font-bold display-font mb-4">1. The On-Device Principle</h2>

                    <p class="text-slate leading-relaxed mb-4">

                        The defining security feature of AdvisorNote is that the entire speech-to-text model runs locally on your PC or Mac. This eliminates the largest data vulnerability associated with cloud-based transcription.

                    </p>

                    <ul class="list-disc pl-5 text-slate space-y-2 leading-relaxed">

                        <li>**No Internet Dependency:** Transcription works completely offline.</li>

                        <li>**Zero Transmission Risk:** Audio files and sensitive transcripts are never sent to external servers.</li>

                        <li>**Data Ownership:** Data remains 100% on the user's local machine, governed by the firm's local IT policies.</li>

                    </ul>

                </div>


                <div class="bg-white p-8 border-2 border-charcoal">

                    <h2 class="text-3xl font-bold display-font mb-4">2. Compliance Focus (Australia)</h2>

                    <p class="text-slate leading-relaxed mb-4">

                        Built specifically for Australian financial service providers, AdvisorNote is designed to simplify adherence to strict local data laws.

                    </p>

                    <ul class="list-disc pl-5 text-slate space-y-2 leading-relaxed">

                        <li>**Privacy Act 1988 (Cth):** By keeping data local, you maintain direct control required for compliance.</li>

                        <li>**Data Sovereignty:** No client data is subject to foreign jurisdictions or US cloud security laws.</li>

                        <li>**Audit Trail:** Transcriptions are saved locally as text files, easily integrated into existing audit and record-keeping systems.</li>

                    </ul>

                </div>


                <div class="bg-white p-8 border-2 border-charcoal">

                    <h2 class="text-3xl font-bold display-font mb-4">3. Application Integrity</h2>

                    <p class="text-slate leading-relaxed mb-4">

                        We employ best practices to ensure the integrity and trust of the application itself.

                    </p>

                    <ul class="list-disc pl-5 text-slate space-y-2 leading-relaxed">

                        <li>**Signed Application:** The installer and application files are digitally signed to guarantee they have not been tampered with since they left our development environment.</li>

                        <li>**Minimal Permissions:** The application only requires necessary permissions to access the microphone and local file system. It has no need for network or root access.</li>

                        <li>**Transparent Architecture:** We provide documentation detailing the Software's architecture to allow for internal IT security and due diligence reviews.</li>

                    </ul>

                </div>


                <div class="bg-white p-8 border-2 border-charcoal">

                    <h2 class="text-3xl font-bold display-font mb-4">4. Summary of Data Movement</h2>

                    <ul class="list-disc pl-5 text-slate space-y-2 leading-relaxed">

                        <li>**Audio/Transcripts:** Stays on device.</li>

                        <li>**License Check:** Encrypted connection to our licensing server to verify the annual license key. This data is non-identifying.</li>

                        <li>**Updates:** Secure HTTPS connection to download application updates.</li>

                        <li>**Transparent architecture:** Transparent architecture for security audits</li>

                    </ul>

                </div>
