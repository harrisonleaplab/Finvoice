Security Overview

How AdvisorNote ensures compliance and data safety.

1. The On-Device Principle

The defining security feature of AdvisorNote is that the entire speech-to-text model runs locally on your PC or Mac. This eliminates the largest data vulnerability associated with cloud-based transcription.

  • **No Internet Dependency:** Transcription works completely offline.
  • **Zero Transmission Risk:** Audio files and sensitive transcripts are never sent to external servers.
  • **Data Ownership:** Data remains 100% on the user's local machine, governed by the firm's local IT policies.

2. Compliance Focus (Australia)

Built specifically for Australian financial service providers, AdvisorNote is designed to simplify adherence to strict local data laws.

  • **Privacy Act 1988 (Cth):** By keeping data local, you maintain direct control required for compliance.
  • **Data Sovereignty:** No client data is subject to foreign jurisdictions or US cloud security laws.
  • **Audit Trail:** Transcriptions are saved locally as text files, easily integrated into existing audit and record-keeping systems.

3. Application Integrity

We employ best practices to ensure the integrity and trust of the application itself.

  • **Signed Application:** The installer and application files are digitally signed to guarantee they have not been tampered with since they left our development environment.
  • **Minimal Permissions:** The application only requires necessary permissions to access the microphone and local file system. It has no need for network or root access.
  • **Transparent Architecture:** We provide documentation detailing the Software's architecture to allow for internal IT security and due diligence reviews.

4. Summary of Data Movement

  • **Audio/Transcripts:** Stays on device.
  • **License Check:** Encrypted connection to our licensing server to verify the annual license key. This data is non-identifying.
  • **Updates:** Secure HTTPS connection to download application updates.
  • **Transparent architecture:** Transparent architecture for security audits
